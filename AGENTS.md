# FoundProof - AGENTS.md

This file defines the permanent rules for the FoundProof project.

All agents, assistants, and automated tools must follow these rules.

---

# Project Definition

FoundProof is a mobile application that records the moment someone finds a lost item and anchors that record as verifiable proof on the Symbol blockchain.

The goal is to transform a finding event into a verifiable record.

Core concept:

"Turn the moment of finding into proof."

---

# What FoundProof IS

FoundProof is:

- a **recording system for finding events**
- a **proof layer for those records**
- a **searchable discovery network for lost items**
- a **tool to support handoff to appropriate institutions**

Examples of institutions:

- police boxes
- station counters
- airport counters
- hotel front desks
- facility reception desks
- event staff
- school offices
- office receptions

---

# What FoundProof IS NOT

FoundProof must NEVER become:

- a peer-to-peer marketplace
- a resale platform
- a person-to-person exchange system
- a classified ads system
- a direct meeting platform

The system must NOT connect individuals directly.

Lost items must be handed over to **institutions**, not individuals.

---

# Blockchain Design Rule

Symbol blockchain is used ONLY as a **proof layer**.

It must not be used as:

- a search database
- a storage system for images
- an NFT ownership platform

Images must remain off-chain.

Only **hashes and minimal metadata** are anchored on-chain.

---

# NFT Policy

Do NOT design the system around NFTs.

The project proves a **finding event**, not **ownership of an item**.

Therefore:

- No NFT minting
- No token ownership logic
- No asset trading

---

# Data Architecture

Responsibilities must be split clearly.

Symbol Blockchain
- proof layer
- record hash anchor
- timestamp integrity

Off-chain database
- search
- indexing
- images
- metadata display

Storage
- original images
- thumbnails

---

# AI Usage

AI may assist with:

- object recognition
- category suggestion
- short description generation
- discovery assistance

AI must NOT:

- determine ownership
- determine legality
- determine theft
- make legal judgments

---

# Security and Safety

The system must:

- avoid exposing precise personal location data
- avoid personal identification
- avoid enabling direct contact between strangers
- prioritize safety and institutional handoff

---

# UI Principles

The interface should communicate:

- trust
- calmness
- public-service feeling
- simplicity
- clarity

Avoid gamification or marketplace-like UI.

---

# Core Principle

FoundProof does NOT prove ownership.

FoundProof proves:

"When and where a person documented finding an item."

---

# End of rules
