# MVP Flow Checklist

## Goal

Confirm the core flow remains stable:

1. capture
2. record
3. hash
4. anchor
5. proof view

## Mock mode

Use this as the default MVP validation path.

### Expected result

- image can be selected or captured
- record can be saved
- `proofRecordHash` exists
- mock anchor succeeds
- proof view shows record hash and tx hash
- search can find the saved record
- runtime indicates `mode=mock`

### UI checklist

- Capture screen opens
- image preview appears
- location capture works or fails gracefully
- AI assist does not block save
- save record succeeds
- proof screen opens
- anchor action succeeds
- detail screen shows:
  - record hash
  - transaction hash
  - explorer link
- search screen returns the created record
- proof presentation remains consistent through the mock flow

## API mode

Use this when the optional real server-side Symbol path is configured.

### Expected result

- record is created via local API
- `proofRecordHash` is generated by the server
- Symbol Testnet anchor succeeds
- `proofTxHash` and explorer link are stored
- detail screen displays on-chain proof reference

### UI checklist

- runtime indicates `mode=api`
- Capture screen save succeeds
- Proof screen anchor succeeds
- Detail screen shows:
  - record hash
  - transaction hash
  - explorer link
- explorer opens Symbol Testnet transaction
- search returns the created record
